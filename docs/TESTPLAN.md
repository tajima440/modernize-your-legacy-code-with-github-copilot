# テストプラン - 学生アカウント管理システム

このドキュメントは、COBOLで実装された学生アカウント管理システムの包括的なテストプランです。
このテストプランは、Node.js への移行時の単体テストおよび統合テストの基準となります。

## テスト実行ガイドライン

### テスト環境
- **システム**: Linux Ubuntu 22.04.5 LTS
- **COBOL コンパイラ**: GnuCOBOL (cobc)
- **コンパイル方法**: `cobc -x src/cobol/main.cob src/cobol/operations.cob src/cobol/data.cob -o accountsystem`
- **実行方法**: `./accountsystem`

### テスト実施順序の推奨
1. TC-001 ～ TC-004：システム基礎テスト
2. TC-005 ～ TC-008：残高表示機能
3. TC-009 ～ TC-015：入金機能
4. TC-016 ～ TC-022：出金成功ケース
5. TC-023 ～ TC-028：出金失敗ケース
6. TC-029 ～ TC-034：複合操作
7. TC-035 ～ TC-040：エッジケース
8. TC-041 ～ TC-045：データ整合性
9. TC-046 ～ TC-050：UI検証
10. TC-051 ～ TC-053：パフォーマンス

### テスト結果の記録
- **Actual Result** に実際の結果を詳細に記入
- **Status** を「Pass」または「Fail」で判定
- **Comments** に問題・懸念事項を記入

### Node.js 移行時の注意点
このテストプランは、以下の要素を Node.js アプリケーションでも同様にテストすることを想定しています：
- 入出金ロジックの正確性
- 残高チェック機能
- データ永続化
- ユーザーインターフェース
- エラーハンドリング

すべてのテストケースが Node.js 版でも実行可能であることを確認してください。
