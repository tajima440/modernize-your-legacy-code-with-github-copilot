# Student Account Management System - Node.js Version

Node.jsへ移行された学生アカウント管理システムです。
元のCOBOLアプリケーションのビジネスロジック、データ整合性、およびメニューオプションを完全に保持しています。

## 概要

このアプリケーションは、元の3つの独立したCOBOLプログラムを単一のNode.jsアプリケーションに統合しました：

### 元のCOBOLアーキテクチャ
- **main.cob (MainProgram)**: ユーザーインターフェース層
- **operations.cob (Operations)**: ビジネスロジック層
- **data.cob (DataProgram)**: データアクセス層

### Node.jsアーキテクチャ
- **DataProgram クラス**: データ層 - 残高の永続化管理
- **Operations クラス**: ビジネスロジック層 - アカウント操作ロジック
- **MainProgram クラス**: プレゼンテーション層 - ユーザーインターフェース

## インストール

### 前提条件
- Node.js 14.0.0 以上
- npm

### セットアップ

```bash
# src/accounting ディレクトリに移動
cd src/accounting

# 依存関係をインストール（既にインストール済みの場合は不要）
npm install
```

## 使用方法

### アプリケーション実行

```bash
# src/accounting ディレクトリから実行
npm start

# または直接実行
node index.js
```

### メニューオプション

アプリケーション起動時に以下のメニューが表示されます：

```
--------------------------------
Account Management System
1. View Balance
2. Credit Account
3. Debit Account
4. Exit
--------------------------------
```

#### 1. View Balance（残高表示）
現在のアカウント残高を表示します。
```
Enter your choice (1-4): 1
Current balance: 001000.00
```

#### 2. Credit Account（入金）
アカウントに金額を加算します。

```
Enter your choice (1-4): 2
Enter credit amount: 500.00
Amount credited. New balance: 001500.00
```

#### 3. Debit Account（出金）
アカウントから金額を差し引きます。残高チェック機能付き：

**成功ケース:**
```
Enter your choice (1-4): 3
Enter debit amount: 300.00
Amount debited. New balance: 001200.00
```

**失敗ケース（残高不足）:**
```
Enter your choice (1-4): 3
Enter debit amount: 2000.00
Insufficient funds for this debit.
```

#### 4. Exit（終了）
アプリケーションを終了します。

## ビジネスロジック

### 初期状態
- **初期残高**: 1000.00

### 入金処理（CREDIT）
1. ユーザーに入金額を入力させる
2. データ層から現在の残高を読み込み
3. 入金額を加算
4. 更新された残高をデータ層に保存
5. 新しい残高をユーザーに表示

### 出金処理（DEBIT）
1. ユーザーに出金額を入力させる
2. データ層から現在の残高を読み込み
3. **残高チェック**: 出金額が現在の残高以下か確認
   - ✓ 成功: 出金を実行、残高を更新
   - ✗ 失敗: エラーメッセージを表示、残高は変わらず

### 残高表示（TOTAL）
1. データ層から現在の残高を読み込み
2. フォーマット済みの残高を表示

## VS Code デバッグ

このプロジェクトには`.vscode/launch.json`が設定されており、VS Codeからの直接デバッグが可能です。

### デバッグ起動

**方法 1: 通常実行**
- VS Code の「デバッグ」メニュー → 「Node.js Accounting App」を選択
- または `F5` キーを押す

**方法 2: デバッグ詳細モード**
- VS Code の「デバッグ」メニュー → 「Node.js Accounting App (Inspect)」を選択
- Chrome DevTools で詳細なデバッグが可能

### デバッグのポイント

- ブレークポイントを設定してステップバイステップ実行が可能
- ウォッチ式でコンソール出力を監視
- 変数の値をリアルタイムで確認

## COBOLからの移行について

### 保持されるロジック
- ✓ 3層アーキテクチャ（データ層、ビジネスロジック層、プレゼンテーション層）
- ✓ initial balance = 1000.00
- ✓ 残高チェック機能（出金時）
- ✓ メニュー方式のユーザーインターフェース
- ✓ データ整合性

### 改善点

- 依存性注入によるテスト可能性の向上

- より詳細なコメントとドキュメンテーション

## トラブルシューティング


- `Ctrl+C` でアプリケーションを停止

### ポート衝突エラー
このアプリケーションはコマンドラインアプリケーションのため、ポート衝突の心配はありません。
